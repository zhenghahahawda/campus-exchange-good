## 接口权限说明

### 权限级别

1. **公开接口**：无需登录即可访问
2. **需登录**：需要有效的用户Token
3. **管理员**：需要管理员权限（通过 `@RequireAdmin` 注解标识）

### 认证方式

- 使用 JWT Token 进行身份验证
- Token 通过请求头传递
- 拦截器 `AuthInterceptor` 负责验证Token有效性
- 管理员权限通过 `PermissionAspect` 切面进行验证

### 跨域配置

部分控制器已配置 CORS 支持：
- `@CrossOrigin(origins = "*", maxAge = 3600)`
- 支持的控制器：商品管理、商品分类管理、订单管理、违规举报管理

## 附录

### 常用状态码

- `200`: 成功
- `400`: 请求参数错误
- `401`: 未授权（未登录或Token无效）
- `403`: 禁止访问（权限不足）
- `404`: 资源不存在
- `500`: 服务器内部错误

### 响应格式

所有接口统一使用以下响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

或

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```
