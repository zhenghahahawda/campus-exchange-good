## 3. 工具类封装

### 3.1 constants.js - 常量管理

**作用**: 统一管理项目中的常量，避免魔法字符串

**包含内容**:
- 商品状态常量 (`GOOD_STATUS`)
- 排序选项常量 (`SORT_OPTIONS`)
- 配送方式常量 (`DELIVERY_TYPES`)
- 商品分类常量 (`GOOD_CATEGORIES`)
- 消息类型常量 (`MESSAGE_TYPES`)

**优势**:
- 避免硬编码字符串
- 统一管理，易于维护
- 支持 IDE 自动补全
- 减少拼写错误

**使用示例**:
```javascript
import { GOOD_STATUS, SORT_OPTIONS } from '@/utils/constants';

// 使用常量而非字符串
if (status === GOOD_STATUS.PENDING) {
  // ...
}
```

### 3.2 goodsManager.js - 商品管理工具类

**作用**: 封装商品相关的业务逻辑

**包含类**:
1. `GoodsFilter` - 商品筛选
   - `filterByStatus()` - 按状态筛选
   - `filterBySearch()` - 按关键词搜索
   - `filter()` - 综合筛选

2. `GoodsSorter` - 商品排序
   - `sort()` - 排序商品列表

3. `GoodsStats` - 商品统计
   - `calculate()` - 计算各状态商品数量

4. `GoodsOperations` - 商品操作
   - `approve()` - 审核通过
   - `reject()` - 驳回商品
   - `toggleShelf()` - 切换上下架
   - `delete()` - 删除商品
   - `update()` - 更新商品

**优势**:
- 业务逻辑与组件解耦
- 易于单元测试
- 可在多个组件中复用
- 代码更清晰易读

### 3.3 messageHelper.js - 消息助手

**作用**: 统一管理消息提示和确认对话框

**包含类**:
1. `MessageHelper` - 消息提示
   - `success()` - 成功消息
   - `warning()` - 警告消息
   - `error()` - 错误消息
   - `info()` - 信息消息

2. `ConfirmHelper` - 确认对话框
   - `confirm()` - 通用确认
   - `confirmDelete()` - 删除确认
   - `confirmApprove()` - 审核确认

**优势**:
- 统一的消息提示接口
- 减少重复代码
- 易于全局样式调整

**使用示例**:
```javascript
import { MessageHelper, ConfirmHelper } from '@/utils/messageHelper';

// 显示成功消息
MessageHelper.success(this, '操作成功');

// 显示删除确认
await ConfirmHelper.confirmDelete(this, '该商品');
```

### 3.4 themeManager.js - 主题管理工具类

**作用**: 管理主题切换和筛选

**包含类**:
1. `ThemeManager` - 主题管理
   - `applyTheme()` - 应用主题
   - `getCurrentTheme()` - 获取当前主题
   - `transitionTheme()` - 主题切换动画

2. `ThemeFilter` - 主题筛选
   - `filterByCategory()` - 按分类筛选
   - `filterBySearch()` - 按关键词搜索
   - `updateCategoryCounts()` - 更新分类计数

3. `Paginator` - 分页工具
   - `paginate()` - 分页处理

## 4. 组件优化

### 4.1 组件拆分原则

1. **单一职责**: 每个组件只负责一个功能
2. **高内聚低耦合**: 组件内部逻辑紧密，组件间依赖少
3. **可复用性**: 组件设计考虑复用场景
4. **Props 明确**: 清晰定义输入输出

## 6. 性能优化

### 6.1 计算属性缓存

使用计算属性自动缓存筛选和排序结果：

```javascript
computed: {
  filteredGoods() {
    let filtered = GoodsFilter.filter(
      this.goodsList, 
      this.filterStatus, 
      this.searchQuery
    );
    return GoodsSorter.sort(filtered, this.sortBy);
  }
}
```

### 6.2 Object.freeze() 优化

冻结不可变数据，减少响应式开销：

```javascript
data() {
  return {
    themes: Object.freeze([...themes])
  }
}
```

### 6.3 组件懒加载

只渲染当前页的数据（分页）：

```javascript
// 每页只渲染 9 个主题卡片
paginatedThemes() {
  const start = (this.currentPage - 1) * this.pageSize;
  const end = start + this.pageSize;
  return this.filteredThemes.slice(start, end);
}
```

## 7. 代码规范

### 7.1 命名规范

- **组件名**: PascalCase (如 `ThemeCard.vue`)
- **文件名**: kebab-case (如 `theme-manager.js`)
- **常量**: UPPER_SNAKE_CASE (如 `GOOD_STATUS`)
- **函数**: camelCase (如 `filterByStatus`)
- **类名**: PascalCase (如 `GoodsFilter`)

### 7.2 注释规范

```javascript
/**
 * 按状态筛选商品
 * @param {Array} goods - 商品列表
 * @param {string} status - 状态
 * @returns {Array} 筛选后的商品列表
 */
static filterByStatus(goods, status) {
  // ...
}
```

### 7.3 导入顺序

```javascript
// 1. Vue 核心
import Vue from 'vue';

// 2. 第三方库
import axios from 'axios';

// 3. 组件
import ThemeCard from '@/components/theme/ThemeCard.vue';

// 4. 工具类
import { GoodsFilter } from '@/utils/goodsManager';

// 5. 常量
import { GOOD_STATUS } from '@/utils/constants';

// 6. 样式
import '@/assets/css/main.scss';
```
